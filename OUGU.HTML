Own-Up Grown-Up 2.0 Explained:

My process in developing this app:

This was the final group project of my Web Development Bootcamp. We had a list of criteria the app must meet, and a group of 4 people to make it happen. Our group decided to build a program that is a reminder system for all of the grown-up tasks that happen so infrequently that it's always a struggle to remember them. We built an app that has a pre-made database of tasks and users can sign up to get reminders texted to remind them to do those tasks on certain days and at certain frequencies.

Technologies Used:

● ReactJS
● CSS3
● RubyOnRails
● PostgreSQL

Barriers to Development:

This was our first true project that we took from concept to deployment. We struggled with a few things along the way. Thinking through the entire MCV and properly structuring the frontend and backend to properly communicate was a very enlightening project. We ran into some problems in getting true backend authentication for our app.

Another big barrier was integrating the external API for the texting service. We found we had to use a worker service to delay sending information to the API in order for the texts to send at the proper times. Since the project, Andy and I have continued to add features and refactor and clean up the code.

There was also a steep learning curve for us with using GitHub. We learned a lot about branches, code review, and merging. This was one of first times we had used GitHub for group projects.

Refactoring:

The first refactor came in changing the way we handled some information and changing our tables around in order to allow the API to have all the information available in one place.

Our next refactor was to add the "Settings" view and allow a user to change their name and phone number as well as preferences settings.

We also added a "Instructions" view to help our users see a very detailed breakdown of what each page does and how to work our app.
